# Frontend для студенческого портала

## Установка и запуск

### 1. Установите зависимости

```bash
cd frontend
npm install
```

### 2. Запустите сервер

```bash
npm start
```

Сервер запустится на `http://localhost:3000`

## Использование

### Авторизация

1. Откройте `http://localhost:3000/login`
2. Введите логин и пароль пользователя с ролью **"student"**
3. После успешного входа вы будете перенаправлены на страницу курсов

### Доступные страницы

- **Авторизация**: `/login`
- **Список курсов**: `/courses`
- **Страница курса**: `/course?id=1`
- **Страница задания**: `/task?id=1`

## Настройка

### Backend URL

По умолчанию backend находится на `http://192.168.56.102/local/api/index.php`

Чтобы изменить URL backend, отредактируйте файл `public/js/api.js`:

```javascript
constructor() {
    this.baseUrl = 'http://192.168.56.102/local/api/index.php';
}
```

## Защита страниц

Все страницы (кроме `/login`) автоматически проверяют авторизацию. Если пользователь не авторизован, происходит редирект на `/login`.

## Выход из системы

На странице курсов есть кнопка "Выйти", которая вызывает API метод `Auth::logout` и перенаправляет на страницу входа.

## API методы

- `Auth::login(login, password)` - авторизация
- `Auth::logout()` - выход из системы
- `User::get()` - получение информации о текущем пользователе
- И другие методы для работы с курсами, лекциями и заданиями

## Отладка

Если возникают проблемы:

1. Проверьте, что backend сервер доступен: `http://192.168.56.102/local/api/index.php`
2. Проверьте консоль браузера (F12) на наличие ошибок
3. Проверьте вкладку Network для просмотра запросов к API
4. Убедитесь, что вы используете правильный логин и пароль
